# 学习运营机制（让进度可持续、可追踪、可纠错）

## 1. 我如何在“每次进入会话”时识别你的进度

每次新会话开始，先执行固定的 `3 步启动流程`：

1. 打开 [学习者档案](./learner_profile.md) 确认当前等级、里程碑和长期目标。
2. 打开 [进度追踪](./05_progress_tracker.md) 读取最近 `3` 次记录，提取：
   - 最近主题与得分趋势
   - 当前薄弱点
   - 上次约定的 `Next Action`
3. 用一句话和你对齐：
   - 「你目前在 X 阶段，最近卡在 Y，本节先完成 Z。」

> 这样即使会话上下文丢失，也能靠文档恢复教学状态。

## 2. 记录规范（避免“学了但没沉淀”）

每次课结束必须同步写三处：

- `05_progress_tracker.md`：记录本节分数、结论、下一步行动。
- `09_error_notebook.md`：记录本节出现的高价值错误（概念误区/实现坑/排障盲区）。
- `learner_profile.md`：仅在阶段变化时更新（例如 L1 -> L2、里程碑完成）。

## 3. 错题本机制（Error Notebook）

错题不是“记答案”，而是沉淀“错误模式”。每条错误至少包含：

- 错误现象（你当时的说法/代码行为）
- 根因分类（概念、实现、调参、评估、工程权衡）
- 正确范式（一句话原则 + 一个可复用检查清单）
- 复练计划（何时、用什么题型复测）
- 状态（Open / Fixed / Regressed）

复习节奏：

- `24h` 复练一次
- `7d` 再复练一次
- 若再次出错，状态标记为 `Regressed` 并加入下节必做题

## 4. 升级的通过门槛（防止“虚高掌握”）

在原有四维评分基础上，新增两条“晋级闸门”：

- **迁移闸门**：同类问题在新场景下也能独立完成。
- **抗错闸门**：错题本中对应错误连续 `2` 次复测通过。

只有两条闸门都通过，才升级到下一层级。

## 5. 我们的协作约定（默认执行）

- 每节开场先回放上次 `Next Action`，不直接开新课。
- 若本节出现重复错误，优先进入 `Debug Clinic` 而不是继续加新知识。
- 每完成一个 Level，产出一个“可演示成果”（脚本、报告或小系统）。
