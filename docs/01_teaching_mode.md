# 教学模式（你和我如何协作）

## 目标

- 不是“听懂”，而是“能做出来、能解释清楚、能评估好坏”。
- 以工程落地为主线：理论最小充分、实践最大有效。

## 教学节奏（每个知识点）

1. `3-8 分钟`：快速建模（核心概念 + 边界）
2. `15-40 分钟`：动手实现（代码/实验）
3. `10-20 分钟`：结果分析（为什么这样、哪里会错）
4. `5-10 分钟`：测评（口头 + 代码 + 设计）
5. `2-5 分钟`：记录进度与下一步

## 难度自适应策略

- 先做“探针题”判断你是否已掌握。
- 你已掌握：跳过细讲，直接做中高阶题或工程题。
- 你不熟悉：补理论图景 + 最小可运行例子 + 反例。

## 课程形态

- `Concept Brief`：快速概念课（高密度）
- `Build Lab`：实现课（代码和实验）
- `System Design`：方案设计课（权衡与架构）
- `Debug Clinic`：问题排查课（真实故障）
- `Paper to Practice`：论文到工程课
