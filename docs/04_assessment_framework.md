# 测评机制（讲完一个点如何判断掌握）

## 1. 四维评分（每维 0-5）

- `Concept`：概念准确性（是否说得清）
- `Implementation`：实现能力（是否能做出来）
- `Debugging`：排障能力（是否能定位并修复）
- `Trade-off`：权衡能力（是否会做工程取舍）

## 2. 掌握等级

- `L0` 不了解：概念不稳定，无法独立实现
- `L1` 入门：可跟做，独立性弱
- `L2` 可用：可独立实现基础版本
- `L3` 熟练：可优化、可解释、可排障
- `L4` 进阶：可做架构与评估体系

## 3. 每节通过标准

- 四维平均分 `>= 3.0`
- `Implementation >= 3`
- 能在新样例上复用（迁移测试通过）

## 4. 题型比例

- 20% 概念快问快答
- 50% 代码和实验
- 30% 场景化系统设计

## 5. 复盘规则

- 低于阈值：下节先补薄弱点，再进入新内容
- 连续两节高分：加速进入更高层知识点
